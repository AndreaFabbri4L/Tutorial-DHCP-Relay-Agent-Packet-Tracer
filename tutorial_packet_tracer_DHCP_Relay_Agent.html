Obiettivi & Prerequisiti

Al termine di questo tutorial sarai in grado di:

  • Progettare e costruire una topologia a due reti in Packet Tracer con router, switch e PC.
  • Configurare un server DHCP che assegni indirizzi dinamici a tutti i client della rete.
  • Configurare le interfacce del router Cisco 2911 con indirizzamento IPv4.
  • Abilitare il DHCP relay (ip helper-address) per far raggiungere il DHCP server ai client di una subnet remota.
  • Verificare la corretta assegnazione degli indirizzi con strumenti come ipconfig e ping.

Prerequisiti

Prima di iniziare assicurati di conoscere i seguenti concetti:

  • Fondamenti del modello TCP/IP e indirizzi IPv4 con notazione CIDR.
  • Concetto di subnet, indirizzo di rete e broadcast.
  • Funzionamento del protocollo DHCP (DORA: Discover, Offer, Request, Acknowledge).
  • Interfaccia base di Cisco Packet Tracer (aggiunta dispositivi, cablaggio, CLI).
🔵 Nota sulle versioni
Il tutorial è stato sviluppato con Cisco Packet Tracer 8.x. Le interfacce grafiche possono variare leggermente nelle versioni precedenti, ma i comandi CLI rimangono identici.
01

Topologia di rete

La rete da realizzare è composta da due subnet separate e connesse da un unico router. Il server DHCP si trova sulla Rete 1 e serve dinamicamente i client di entrambe le reti grazie al relay configurato sull'interfaccia del router verso la Rete 2.

Piano di indirizzamento

Dispositivo Interfaccia Indirizzo IP Subnet Mask Gateway
DHCP Server FastEthernet0 192.168.10.2 255.255.255.0 192.168.10.1
Router 2911 GigabitEthernet0/0 192.168.10.1 255.255.255.0 N/A
Router 2911 GigabitEthernet0/1 192.168.20.1 255.255.255.0 N/A
PC Rete 1 NIC Assegnato via DHCP 255.255.255.0 192.168.10.1
PC Rete 2 NIC Assegnato via DHCP 255.255.255.0 192.168.20.1
02

Creazione del progetto in Packet Tracer

  1. Apri Cisco Packet Tracer.
  2. Dal menu File seleziona Save As...
  3. Salva il file con il nome: DHCP_Relay_[TuoCognome].pkt (es: DHCP_Relay_Rossi.pkt).
💾 Ricorda di salvare spesso
Durante l'esercitazione premi periodicamente Ctrl+S (o Cmd+S su Mac) per non perdere i progressi in caso di crash del simulatore.
03

Aggiunta dei dispositivi al workspace

Tutti i dispositivi si trovano nella barra in basso di Packet Tracer, suddivisi per categoria. Trascina ciascun dispositivo nell'area di lavoro posizionandolo in modo da rispettare la topologia mostrata nella sezione 01.

Dispositivi da aggiungere

DispositivoCategoria PTModelloQuantità
RouterNetwork Devices → Routers29111
SwitchNetwork Devices → Switches2960 (o generico)2
PCEnd Devices → End DevicesPC generico4
Server DHCPEnd Devices → End DevicesServer generico1
  1. Seleziona la categoria Routers nella barra in basso a sinistra, poi trascina un 2911 al centro del workspace.
  2. Seleziona la categoria Switches e trascina due switch: posiziona Switch 1 a sinistra del router e Switch 2 a destra.
  3. Seleziona End Devices e trascina 4 PC: 2 collegati logicamente alla Rete 1 (sinistra) e 2 alla Rete 2 (destra).
  4. Sempre da End Devices trascina un Server e posizionalo vicino allo Switch 1 (Rete 1).
  5. Rinomina i dispositivi facendo doppio clic sull'etichetta: usa i nomi Router0, Switch1, Switch2, PC-1, PC-2, PC-3, PC-4, DHCP-Server.
04

Cablaggio della rete

I collegamenti tra dispositivi differenti (es. PC-Switch o Switch-Router) avvengono con un cavo dritto in rame (Copper Straight-Through), contrassegnato dall'icona della linea nera continua.

  1. Clicca sull'icona delle Connections (il fulmine arancione).
  2. Seleziona il Copper Straight-Through (linea nera continua).
  3. Collega PC-1 e PC-2 allo Switch 1 (scegliendo le porte FastEthernet).
  4. Collega il DHCP-Server allo Switch 1.
  5. Collega PC-3 e PC-4 allo Switch 2.
  6. Collega lo Switch 1 alla porta GigabitEthernet0/0 del Router.
  7. Collega lo Switch 2 alla porta GigabitEthernet0/1 del Router.
  8. Attendi che i led sulle porte dello switch diventino verdi (puoi accelerare il tempo premendo Alt+W o il tasto Fast Forward Time in basso). Nota: I led lato router rimarranno rossi finché le porte non verranno accese tramite configurazione CLI. Se rimangono arancioni, attendi qualche secondo; se rimangono rossi controlla il tipo di cavo o la porta selezionata.
05

Configurazione del Server DHCP

Il server DHCP ha due compiti: possedere un IP statico e distribuire indirizzi dinamici ai client di entrambe le reti. In Packet Tracer il server si configura attraverso la GUI grafica.

5.1 – Assegnazione IP statico al server

  1. Fai doppio clic su DHCP-Server per aprire la finestra di configurazione.
  2. Vai nella scheda Desktop → IP Configuration.
  3. Seleziona Static e inserisci i seguenti valori:
CampoValore da inserire
IP Address [ 192.168.10.2 ]
Subnet Mask [ 255.255.255.0 ]
Default Gateway [ 192.168.10.1 ]

5.2 – Configurazione del servizio DHCP

  1. Sempre nella finestra del server, vai nella scheda Services.
  2. Nel menu a sinistra, seleziona DHCP.
  3. Assicurati che il servizio sia su On.
  4. Crea il primo pool (per la Rete 1): modifica il pool di default (spesso chiamato serverPool) o creane uno nuovo chiamato POOL_RETE1. Inserisci i dati e clicca Save o Add.
  5. Crea il secondo pool (per la Rete 2) chiamato POOL_RETE2, compila i dati e clicca su Add.

Dati per POOL_RETE1

Pool Name [ POOL_RETE1 ]
Default Gateway [ 192.168.10.1 ]
Start IP Address [ 192.168.10.16 ]
Subnet Mask [ 255.255.255.0 ]
Maximum number of Users [ 16 ]

Dati per POOL_RETE2

Pool Name [ POOL_RETE2 ]
Default Gateway [ 192.168.20.1 ]
Start IP Address [ 192.168.20.16 ]
Subnet Mask [ 255.255.255.0 ]
Maximum number of Users [ 16 ]
✅ Perché due pool?
Il server DHCP identifica la subnet del client richiedente tramite l'IP sorgente della richiesta (o tramite il relay agent IP). Avendo due subnet distinte, occorrono due pool separati con i rispettivi gateway predefiniti. Il pool corretto viene selezionato automaticamente in base all'indirizzo di provenienza della richiesta.
06

Configurazione del Router 2911

Il router è il cuore della topologia: instrada il traffico tra le due reti e, grazie al comando ip helper-address, inoltrerà i broadcast DHCP provenienti dalla Rete 2 verso il server DHCP nella Rete 1.

⚠️ Interfacce spente per default
Le interfacce del router Cisco 2911 sono in stato administratively down per impostazione predefinita. Devi attivarle esplicitamente con il comando no shutdown.

6.1 – Accesso alla CLI del router

  1. Fai doppio clic sul router nel workspace.
  2. Clicca sulla scheda CLI. Se compare il messaggio "Would you like to enter the initial configuration dialog?", rispondi no e premi Invio.
  3. Premi Invio per accedere al prompt > (user EXEC mode).

6.2 – Configurazione interfaccia GigabitEthernet0/0 (Rete 1)

Router0 — Configurazione G0/0 (Rete 1) 
Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# description Connessione a Rete 1 - 192.168.10.0/24
Router(config-if)# ip address [ INSERIRE: 192.168.10.1 ] [ INSERIRE: 255.255.255.0 ]
Router(config-if)# no shutdown
! Deve comparire: %LINK-5-CHANGED: Interface GigabitEthernet0/0, changed state to up
Router(config-if)# exit
Router(config)#

6.3 – Configurazione interfaccia GigabitEthernet0/1 (Rete 2)

Router0 — Configurazione G0/1 (Rete 2) 
Router(config)# interface GigabitEthernet0/1
Router(config-if)# description Connessione a Rete 2 - 192.168.20.0/24
Router(config-if)# ip address [ INSERIRE: 192.168.20.1 ] [ INSERIRE: 255.255.255.0 ]
Router(config-if)# no shutdown
! Deve comparire: %LINK-5-CHANGED: Interface GigabitEthernet0/1, changed state to up
Router(config-if)# exit

6.4 – Salvataggio della configurazione

Salvataggio configurazione router 
Router(config)# end
Router# write memory
! oppure: copy running-config startup-config
! Deve comparire: Building configuration... [OK]
07

Configurazione DHCP Relay (ip helper-address)

I messaggi DHCP viaggiano in broadcast (IP di destinazione 255.255.255.255). I router, per regola generale, bloccano i messaggi di broadcast per evitare tempeste di rete. Di conseguenza, i client della Rete 2 non potranno mai raggiungere il server DHCP nella Rete 1 in modo autonomo.

Per risolvere questo problema, dobbiamo istruire l'interfaccia G0/1 (quella della Rete 2) ad agire da Relay Agent, ovvero a catturare i broadcast DHCP, trasformarli in pacchetti unicast (diretti al server specifico) e inoltrarli.

Configurazione IP Helper su G0/1 
Router# configure terminal
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip helper-address [ INSERIRE IP DHCP SERVER: 192.168.10.2 ]
Router(config-if)# end
Router# write memory
🔄 Il flusso logico del DHCP Relay
Quando PC-3 (Rete 2) invia un DHCP Discover in broadcast, il router lo riceve sull'interfaccia G0/1. Grazie a ip helper-address, il router crea un nuovo pacchetto unicast: IP sorgente = G0/1 (192.168.20.1), destinazione = IP del server DHCP (192.168.10.2). Il server risponde al router, che poi smista la risposta al client corretto. Il server capisce a quale pool appartiene la richiesta grazie al campo relay agent IP address nel pacchetto.

Verifica rapida della configurazione router

Verifica interfacce e relay 
Router# show ip interface brief
! Output atteso:
! GigabitEthernet0/0      192.168.10.1    YES manual up                    up
! GigabitEthernet0/1      192.168.20.1    YES manual up                    up
08

Configurazione PC della Rete 1 (PC-1 e PC-2)

I PC della Rete 1 si trovano nello stesso segmento (dominio di broadcast) del server DHCP, quindi otterranno l'IP direttamente, senza bisogno del router.

  1. Fai doppio clic su PC-1.
  2. Vai in Desktop → IP Configuration.
  3. Cambia la selezione da Static a DHCP.
  4. Attendi qualche secondo. Deve comparire la scritta DHCP request successful e verranno assegnati:
    • IP Address: 192.168.10.16 (o successivo)
    • Subnet Mask: 255.255.255.0
    • Default Gateway: 192.168.10.1
  5. Ripeti l'operazione identica per PC-2.
09

Configurazione PC della Rete 2 (PC-3 e PC-4)

I PC della Rete 2 dipendono interamente dal Relay Agent configurato sul router.

  1. Fai doppio clic su PC-3.
  2. Vai in Desktop → IP Configuration e seleziona DHCP.
  3. Attendi l'assegnazione: il PC riceverà un IP nel range 192.168.20.16–192.168.20.31, subnet mask 255.255.255.0, gateway 192.168.20.1.
  4. Ripeti per PC-4.
📌 Cosa succede dietro le quinte
PC-3 invia un broadcast DHCP Discover. Lo Switch 2 lo inonda su tutte le porte. Il router lo riceve su G0/1 e, grazie a ip helper-address, lo incapsula in un pacchetto unicast verso 192.168.10.2. Il server DHCP riceve la richiesta, guarda il campo giaddr (192.168.20.1), seleziona il POOL_RETE2 e risponde con un indirizzo della Rete 2. Il router riceve la risposta e la inoltra a PC-3.
10

Test di connettività e verifica

Affinché la rete possa dirsi completamente funzionante, dobbiamo accertarci non solo che l'IP venga assegnato, ma che i PC delle due reti riescano a parlarsi (grazie al routing del Cisco 2911).

Test A – Verifica con ipconfig

  1. Apri PC-1, vai in Desktop e apri il Command Prompt.
  2. Digita ipconfig e verifica che l'indirizzo IPv4 e il gateway siano corretti per la Rete 1.

Test B – Ping intra-rete (stessa subnet)

  1. Sempre dal Command Prompt di PC-1, digita: ping [IP_DI_PC_2] (es: ping 192.168.10.17).
  2. Verifica la risposta: 4 reply ricevuti, 0% di packet loss.

Test C – Ping inter-rete (tra subnet diverse)

  1. Sempre da PC-1, digita: ping [IP_DI_PC_3] (es: ping 192.168.20.16).
  2. Attenzione: Il primo o i primi due pacchetti potrebbero andare in timeout (Request timed out) mentre i dispositivi risolvono gli indirizzi MAC tramite ARP. I successivi avranno successo.
  3. Ripeti il comando: questa volta avrai 4 risposte su 4.

Test D – Ping al DHCP Server

  1. Dal Command Prompt di PC-3 (Rete 2), digita: ping 192.168.10.2
  2. Il ping deve avere successo: conferma che il server DHCP è raggiungibile anche dalla Rete 2.

Test E – Verifica con Simulation Mode

  1. Imposta uno dei PC della Rete 2 su Static IP (libera un indirizzo del pool) poi rimettilo su DHCP.
  2. Attiva la Simulation Mode (in basso a destra, icona del cronometro).
  3. Filtra per mostrare solo i pacchetti DHCP e UDP.
  4. Premi Play e osserva il percorso dei pacchetti DORA attraverso il router verso il server.
11

Riepilogo finale e concetti chiave

Ottimo lavoro! Hai configurato con successo una rete complessa che simula uno scenario aziendale reale. Ricorda i concetti fondamentali appresi in questo laboratorio:

  • Broadcast Isolation: I router separano i domini di broadcast. Senza un intervento, protocolli basati sul broadcast come il DHCP non possono superare il router.
  • Relay Agent: Il comando ip helper-address trasforma il router in un proxy, consentendo di avere un unico server DHCP centralizzato per dozzine di subnet diverse (risparmio di risorse).
  • Pool Multipli: Il server DHCP è abbastanza "intelligente" da capire quale blocco di IP assegnare guardando da quale subnet (giaddr) arriva la richiesta inoltrata dal router, non l'IP del server.
  • Verifica sistematica: usare ipconfig, ping e la Simulation Mode per confermare ogni singolo aspetto della configurazione.

Domande di verifica

  1. Perché il server DHCP non può avere un indirizzo assegnato dinamicamente dallo stesso pool che gestisce?
  2. Cosa accadrebbe se dimenticassimo di inserire ip helper-address sull'interfaccia G0/1 del router?
  3. Qual è il significato del campo giaddr (Gateway IP Address) nel pacchetto DHCP?
  4. Potresti aggiungere una terza rete alla topologia senza aggiungere un secondo server DHCP? Come?
  5. Cosa cambia nel funzionamento del relay se lo scope DHCP non copre la subnet della Rete 2?
🎯 Estensione opzionale
Prova ad aggiungere un DNS Server statico nella Rete 1 e a configurarne l'IP nei pool DHCP. Poi testa la risoluzione dei nomi aprendo il browser su uno dei PC e navigando verso un indirizzo simbolico.
